Introduction

The Infragistics CAB Extensibility Kit is an add-on to the Infragistics Windows Forms (CLR 4) product. This add-on allows your Infragistics Windows Forms toolset to interoperate with the Microsoft Composite UI Application Block (CAB) framework.

Note: For more details on the Infragistics CAB Extensibility Kit, refer to the "Infragistics CAB Extensibility Kit" section in the Infragistics Windows Forms help.

Assemblies

To build a Windows Forms application using the Infragistics CAB Extensibility Kit, the following assemblies are required:

  • Microsoft CAB Assemblies
    • (Note: Microsoft provides these assemblies for CLR 2.0 only.  In theory, these can be used in any CAB application targeting CLR 4. However, if a developer discovers an incompatability in their CLR 4 implementation, they will need to re-compile the Microsoft source code for these assemblies against CLR 4.)  
    • Microsoft.Practices.CompositeUI
    • Microsoft.Practices.CompositeUI.WinForms
    • Microsoft.Practices.ObjectBuilder
  • Infragistics Assemblies
    • Infragistics4.Shared.vX.X
    • Infragistics4.Win.UltraWinDock.vX.X
    • Infragistics4.Win.UltraWinExplorerBar.vX.X
    • Infragistics4.Win.UltraWinStatusBar.vX.X
    • Infragistics4.Win.UltraWinTabbedMdi.vX.X
    • Infragistics4.Win.UltraWinTabControl.vX.X
    • Infragistics4.Win.UltraWinToolbars.vX.X
    • Infragistics4.Win.UltraWinTree.vX.X
    • Infragistics4.Win.vX.X

To deploy your CAB application, the following assemblies are required:

  • Infragistics.Practices.CompositeUI.WinForms
  • Microsoft.Practices.CompositeUI
  • Microsoft.Practices.CompositeUI.WinForms
  • Microsoft.Practices.ObjectBuilder
  • Any Infragistics assemblies for Infragistics controls/components used in the CAB application

Getting Started
  1. Download the Microsoft Smart Client Software Factory, which includes the Microsoft CAB dlls:
    http://www.microsoft.com/downloads/details.aspx?FamilyId=2B6A10F9-8410-4F13-AD53-05A202FBDB63&displaylang=en
  2. Make sure that you review Microsoft’s documentation and quick-start samples to familiarize yourself with the CAB framework:
    http://msdn2.microsoft.com/en-us/library/aa480450.aspx
  3. Open the Infragistics CompositeUI Solution located in the "[Infragistics_samples_folder]\[version]\CAB Extensibility Kit" folder and compile it.
    Note:
    • [Infragistics_samples_folder] resolves to "C:\Documents and Settings\All Users\Documents\Infragistics" on Windows XP, and "C:\Users\Public\Documents\Infragistics" on Windows Vista.
    • The solution files in the Infragistics CAB Extensibility Kit are designed to use relative references to the Microsoft CAB. If the references are not found, you can remove the references and re-add them from the appropriate location, or you can change the path property on the reference to the appropriate location.
  4. Use the Infragistics CompositeUI Solution in your CAB project by doing either of the following:
    • Open and compile the Infragistics.CompositeUI.WinForms.sln file. This solution outputs the Infragistics.CompositeUI.WinForms.dll assembly. Once you have this assembly compiled, reference it in your CAB project.
    • From your CAB project, reference the projects in the Infragistics.CompositeUI.WinForms.sln file.
  5. Open the BankTeller.sln file to run the Microsoft bank teller quick-start application that was ported to use Infragistics-specific CAB workspaces, UIElementManagers, and controls and components. (This file is located in the "\CAB Extensibility Kit\Samples\BankTeller" folder.) Use this quick-start application to familiarize yourself with the integration of Infragistics Windows Forms and the CAB framework.
  6. Review the Starter Kits provided in the "\CAB Extensibility Kit\Samples" folder. These are a great way for you to become familiar with the product so you can get up and running as quickly as possible.

Ink Support

Our base application class (IGWindowsFormsApplicationBase) includes the ProvideInkSupport property, which is a virtual property that is used to determine whether the shell should create a WinInkProvider for each container control in the application. By default, we will do so on a Tablet PC system when the Tablet PC SDK and our Infragistics.Win.UltraWinInkProvider assembly are installed. This lets the SmartPart developers avoid having to include a dependency on the Infragistics.Win.UltraWinInkProvider assembly; therefore, the shell developer can decide whether or not ink support should be included.
